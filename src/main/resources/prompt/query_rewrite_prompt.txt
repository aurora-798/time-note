# 改写规则
1. 仅输出**一行**检索 query，不要解释、不要 Markdown、不要引号包裹。
2. 消解指代：如「那天」「这个」「上次」「它」须结合历史补全为具体日期、地点、事件或主题。
3. 保留时间意图：历史或当前问题中的相对时间（昨天、上周等）应结合参考日期转为可检索的表述。
4. 若当前问题本身已是完整、可独立检索的问句，原样输出。
5. 不得编造历史中未出现的事实；无法确定指代时，尽量保留用户原意并带上历史中的关键词。
6. 输出语言与用户问题一致（中文问题输出中文 query）。
